# Adaptive Music - Magenta Realtime 2 Test

*English below / [日本語はこちら](#日本語)*

An experiment in turning environmental sound (noise) into music, built on
Google **Magenta RealTime 2** (MLX backend, native on Apple Silicon).

Instead of masking the environment with music, the system amplifies the
"musical interpretation" latent in environmental sound: a live instrument for
balancing the raw recording (reality) against a generated layer (the AI's
reading of it) in real time.

## Layout

- `scripts/live.py` + `live.js` — the live performance system. The UI is built
  entirely in JS (no HTML file)
- `samples/` — environmental recordings (CC/PD sources from Wikimedia Commons;
  `*.ogg`/`*.flac` are the originals, see [CREDITS.md](CREDITS.md))
  - rain_long (thunderstorm, 4 min) / birds_long (dawn chorus, 10 min) /
    ocean_long (lake surf, 5.5 min) / city_long (Mexico City, 5 min, Félix Blume)
  - wav versions are generated by `scripts/prepare_samples.sh` (not in the repo)
- `out/` — session recordings from `--record` (not in the repo)

## Setup / run (macOS, Apple Silicon)

Requirements: Apple Silicon Mac, Python 3.11+ (below uses [uv](https://docs.astral.sh/uv/)
to fetch 3.12 automatically), ffmpeg (`brew install ffmpeg`), ~4 GB disk for model weights.

```bash
git clone https://github.com/AtsuyaTsuchida/adaptive-music-mrt2-test.git
cd adaptive-music-mrt2-test

# environment (plain `python3 -m venv` works too if your python is 3.11+)
curl -LsSf https://astral.sh/uv/install.sh | sh   # if you don't have uv
uv venv --python 3.12 .venv
source .venv/bin/activate
uv pip install "magenta-rt[mlx]==2.0.2" sounddevice==0.5.5

scripts/prepare_samples.sh                        # fetch + convert samples (first run only)
mrt models init && mrt models download mrt2_base  # model weights (~4 GB)

python scripts/live.py --record                   # audio starts after the model loads
open http://localhost:8241                        # control panel
```

Model weights live in `~/Documents/Magenta/magenta-rt-v2/` (override with
`MAGENTA_HOME`). Keep the machine plugged in — on battery the GPU is throttled
below realtime.

## Listening in the browser (remote machines)

Instead of the server's audio device, the **listen here** button on the control
panel plays the mix in the browser: `/stream` serves raw PCM (s16le, 48 kHz
stereo) over chunked HTTP, scheduled for playback with Web Audio. Multiple
listeners are supported; slow clients drop blocks instead of stalling the mix.

Typical remote setup:

```bash
# remote machine: no audio device needed
python scripts/live.py --no-audio --record

# local machine: forward the port
ssh -L 8241:localhost:8241 user@remote
# open http://localhost:8241 and hit "listen here"
```

Machines without an audio device fall back to streaming-only automatically,
even without `--no-audio`.

Alternatively `--host 0.0.0.0` exposes the panel directly on the LAN
(no auth — trusted networks only), reachable at `http://<machine-ip>:8241`.

## Running on Windows (NVIDIA GPU)

`live.py` picks its backend automatically: MLX on macOS, JAX elsewhere
(override with `--backend mlx|jax`). JAX has no GPU support on native Windows,
so use **WSL2 (Ubuntu) + CUDA**:

1. Windows side: NVIDIA driver + WSL2 + Ubuntu (`wsl --install`)
2. Inside WSL:
   ```bash
   git clone https://github.com/AtsuyaTsuchida/adaptive-music-mrt2-test.git && cd adaptive-music-mrt2-test
   python3 -m venv .venv && source .venv/bin/activate
   pip install "magenta-rt[jax]==2.0.2" "jax[cuda12]" sounddevice==0.5.5
   sudo apt install ffmpeg libportaudio2
   scripts/prepare_samples.sh
   mrt models init
   mrt checkpoints download mrt2_base.safetensors   # JAX weights (mrt models download = MLX only)
   python scripts/live.py --record          # backend=jax is selected automatically
   ```
3. Open http://localhost:8241 in a browser (the Windows side is fine)

Notes:
- Audio comes out through WSLg's PulseAudio on Windows 11; check `pactl info`
  if it doesn't
- Realtime generation needs a capable GPU (8 GB+ VRAM recommended). If
  generation falls below realtime, only the generated layer fades out — the
  environmental layer keeps playing
- The JAX backend can take a while on first launch (checkpoint fetch + compile)

## How it works

What you hear is a two-layer mix:

1. **Env layer** — the raw environmental recording, looped
2. **Gen layer** — the MRT2 stream: autoregressive 40 ms frames, with `state`
   carried across chunks so the music never breaks

The generation is conditioned on a weighted average of **MusicCoCa embeddings**
(audio and text share one 768-dim space), which is the heart of the realtime
control:

| parameter | role |
|---|---|
| `w_audio` | blend of env-audio embedding (1.0) vs text prompt (0.0) — the "musicality" axis |
| `env_gain` / `gen_gain` | layer volumes (applied at playback; takes effect in ~0.25 s) |
| `drums` | rhythm conditioning for the gen layer (auto / off / on) |
| `env` | source switch (changes both the env layer and the audio embedding) |
| `prompt` | text prompt (has an effect when w_audio < 1.0) |

### Design notes

- **MLX only allows inference on the thread that loaded the model**
  (`There is no Stream(gpu, 1)`) → the model is loaded inside the generator
  thread
- **Embeddings run in a dedicated worker thread** — a prompt/source change never
  stalls generation; the old style keeps playing until the new embedding is
  ready. All samples are pre-embedded at startup
- **Mixing happens at playback time** — the env layer is plain file playback and
  must never depend on generation keeping up. When generation falls behind,
  only the gen layer fades out and reality carries on
- Generation speed on an M3 Max is 0.85–1.03x realtime depending on machine
  load and power state. **On battery the GPU is throttled below realtime** —
  keep the machine plugged in
- The output device is fixed when the stream opens; switching to AirPods etc.
  requires a restart

## 仕組み

聴こえる音は2レイヤーのミックス:

1. **原音レイヤー** — 環境音の録音そのもの（ループ再生）
2. **生成レイヤー** — MRT2のストリーム生成。40msフレームの自己回帰で、
   `state` をチャンク間で引き継ぐため音楽は途切れない

生成の条件は **MusicCoCa埋め込み**（音声とテキストが同一の768次元空間を共有）の
加重平均で、これがリアルタイム操作の中心:

| パラメータ | 役割 |
|---|---|
| `w_audio` | 環境音の埋め込み(1.0) ⇄ テキストプロンプト(0.0) のブレンド——「音楽度」の軸 |
| `env_gain` / `gen_gain` | レイヤー音量（再生側で適用、反映約0.25秒） |
| `drums` | 生成レイヤーへのリズム条件（auto / off / on） |
| `env` | 環境音ソース切替（原音レイヤーと埋め込みの両方が切り替わる） |
| `prompt` | テキストプロンプト（w_audio < 1.0 で効く） |

### 設計メモ

- **MLXはモデルをロードしたスレッドでしか推論できない**
  （`There is no Stream(gpu, 1)`）→ モデルは生成スレッド内でロード
- **埋め込みは専用ワーカースレッドで計算** — プロンプト/ソース変更で生成は止まらない。
  新しい埋め込みができるまで旧スタイルで演奏継続。全サンプルは起動時に事前埋め込み
- **ミックスは再生側で実施** — 原音レイヤーはただのファイル再生であり、生成の
  遅延に依存させない。生成が遅れた場合は生成レイヤーだけがフェードアウトし、
  現実（環境音）が残る
- 生成速度はM3 Maxで実時間の0.85〜1.03倍（マシン負荷・電源状態に依存）。
  **バッテリー駆動ではGPUが絞られ実時間を割る**ため電源接続を推奨
- 出力デバイスはストリーム開始時に固定。AirPods等への切替は再起動が必要
